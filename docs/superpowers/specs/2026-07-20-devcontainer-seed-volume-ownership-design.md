# Devcontainer seed-volume ownership repair

## Outcome

Make the local seed script generated by `my:project-claude-setup` work on fresh
Docker named volumes and recover existing volumes whose mountpoints are owned by
root. Preserve the current read-only host-seed and writable container-local
volume architecture.

## Design

The generated `local-seed.sh` will repair ownership of both destination
mountpoints before checking `~/.claude/.seeded`. It will use the current
container user's numeric UID and GID with passwordless `sudo`, so a previously
seeded volume also self-heals without repeating copies or installers.

The dotfiles copy will treat the named-volume mountpoint as an existing
destination. When that directory is empty, it will copy the seed directory's
contents into it rather than guard on the destination directory's existence.
Non-empty dotfiles destinations remain untouched.

The seed script will also ensure the container-local `~/.zshrc` contains one
guarded source line for `~/.dotfiles/ai/vekil/env.zsh`. This hook is installed
before the sentinel check so existing seeded containers gain it on their next
start. It loads both the proxy endpoint variables and the managed `codex` shell
function without running the full dotfiles installer or modifying the host.

No Compose init service, root entrypoint, or writable host mount will be added.

## Failure handling

Ownership repair is required and remains under `set -euo pipefail`; failure
stops startup with the underlying `sudo`/`chown` error. Existing non-fatal
marketplace and Codex installer behavior is unchanged. The sentinel is written
only after the copy and installer stages complete.

- a pre-existing sentinel still installing the zsh hook exactly once; and
- a new interactive zsh resolving the Vekil URLs and `codex` wrapper through
